# Data Layout

Committed data is limited to small JSON/JSONL artifacts needed for replay and
debugging. Large data should be placed under `data/raw/`, which is ignored by
git through `.gitignore`.

Recommended layout:

```text
data/raw/
  cityrefer/
    bbox/
      <scene>_bbox.json
    desc/
      CityRefer_val_ND_infer_result.jsonl
      CityRefer_val_NO_infer_result.jsonl
    context_images/
  cityanchor/
    instances/
      <scene>.txt
      <scene>_building.txt
      <scene>_tree.txt
    bbox/
      <scene>_bbox.json
    desc/
      cityanchor_val_ND_infer_result.jsonl
      cityanchor_val_NO_infer_result.jsonl
    context_images/
  sensaturban/
    point_clouds/
    instances/
  sam3/
    cluster_config.yaml
```

Generated outputs:

```text
outputs/
  cityrefer/ND/
  cityrefer/NO/
  cityanchor/ND/
  cityanchor/NO/
  sam3/
```

Do not commit:

- API keys or `.env`
- checkpoints
- PLY / LAS / LAZ / PCD point clouds
- SAM3 masks
- rendered image dumps
- generated bytecode caches
